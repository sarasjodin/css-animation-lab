From Idea to Deployment
Building a Website with Sass 7-1 method and Parcel automation

Key Learnings

  1. I should have started simple (avoid doing the 7-1 method as my first Sass assignment)
  2. Some parts like the variables, nesting and simple mixins are quite intuitive, other things I tested I'll have to improve with time.
  3. We need some time to learn new tools, but working with Scss and Parcel were very intuitive.
  4. I had some issues with paths to urls and images, and put a lot of work in finding the perfect npm run script to clear cache and rebuild quickly. I believed it saved me lots of time - especially in the long run!

Finally I have started my Sass journey ‚ù§Ô∏è I've always been interested, but I never exceeded the work load to get through with a Parcel or similar automation process, without that there is no reason why you should go from css to Sass.

My Development Process

  1. Reused a previous boilerplate (but removed all previous css code and Git history)
  2. Created the 7-1 patters for Sass and its main.scss file with @use
  3. Installed Sass npm install sass -D
  4. Experimented with a Bootstrap menu for large screens Bootstrap Offcanvas responsive menu
  5. Realized that I was already developing and had to start commiting to GitHub...
  6. Created the GitHub repository!
  7. I decided to use Render for the first time for this site. Preeviously I've mainly used Netlify and GitHub Pages for CI/CD deployment.
  8. In the end I decided to create a pure scss site, to avoid Bootstrap dependancy problems for my first scss-site.
  9. I've cheched the requests for the assigment and started to sketch down some ideas for mixins. I decided to start with a mixin handling background colors, hover colors, different shapes (squares and circular shapes) and sizes of social media icons. I was almost able to fulfill the assignment requirement only with my first Mixin...
  10. The next step was to plan and prepare the pages. I decided quickly the pages needed and decided to add some of my old portfolio pages...to create yet another portfolio page!
  11. After working some I realized that my 7-1 method wasn't very well organized. So I started to dived the scss in a more strict way. It felt like I would never finish.
  12. At this point I started to improve the color variables, the responsiveness and the header, hamburger menu and footer. I ended up with a lot of color variables, because I would like to create a reusable and flexible boilerplate (a boilerplated is like a template) to improve with time.
  13. When I learnt to differentiate (mainly variables and mixins in the code) like this:
    @use '../abstracts/mixins' as mixin;
    @use '../abstracts/variables' as vars;
    it got easier to pick up the new syntax.

For this web site I have implemented

  • social icons with shape, size and... variables including if statement
  • light and dark theme with theme button
  • Responsive header with a hamburger menu for mobiles
  • Breakpoint variables for media queries
  • Every second section has another color, to create some variations and structure.
  • Implemented primary and secondary button styles.
  • Started to create some policy templates for the footer links.
  • grid template for my color card tests, see below.

saturate()

Increases the color intensity

$color: saturate(#3498db, 30%);

desaturate()

Reduces color intensity

$color: desaturate(#3498db, 30%);

adjust-hue()

Change the hue by a given degree

$color: adjust-hue(#3498db, 45deg);

rgba()

Adds transparency

$color: rgba(#3498db, 0.5);

mix()

Blends two colors

$color: mix(#3498db, #ff5733, 50%);

invert()

Invert a color

$color: invert(#3498db);
